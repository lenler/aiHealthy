## 项目经历

### 轻食AI｜基于大模型的健康饮食助手（前后端开发负责人） 2026.01 - 2026.03

- 面向个人用户设计健康饮食管理平台，打通“餐食记录（手填/识图）→ 数据入库 → 仪表盘展示 → AI 个性化建议/对话咨询”的核心业务闭环。
- 基于 React + TypeScript + Ant Design + Zustand 构建前端核心模块，覆盖登录注册、仪表盘、饮食分析、对话咨询、个人中心；通过状态持久化降低重复操作成本。
- 封装 Axios 请求层与拦截器，统一 token 注入与异常处理，并在接口层面处理用户或脚本在短时间内多次上传的行为；按业务域拆分 API（登录、饮食、分析、聊天、个人中心），提升接口一致性与可维护性。
- 基于 Express + Sequelize + MySQL 设计并实现后端服务，覆盖用户、健康档案、饮食记录、聊天记录等核心模型；通过业务校验与联合唯一约束保障同用户同日同餐次数据一致性。
- 接入多模型能力：视觉模型用于餐食识别与热量估算，对话模型用于健康问答与流式输出，建议模型用于结构化饮食建议生成，实现 AI 能力与真实业务数据联动。
- 在对话场景中引入工具调用读取当日饮食记录，使 AI 建议从通用回答升级为“基于用户真实数据”的个性化回答。

## 使用说明

- 如果投递方向是前端实习，建议保留“前后端开发负责人”但在面试中强调“前端主导工作占比”。

- 面试口述时按“业务问题 → 技术方案 → 工程保障 → 结果”顺序表达，避免只讲技术栈。
