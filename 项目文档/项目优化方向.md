## Token 处理逻辑优化方向 未完成

### 前端侧
- 统一请求拦截器注入 Authorization: Bearer Token
- 401/403 响应统一处理，清理登录态并跳转登录
- Access Token + Refresh Token 方案，支持静默刷新
- 并发刷新锁，避免多请求同时刷新
- 多标签页同步登出与会话失效

### 后端侧
- 全局鉴权中间件校验 JWT，解析 userId
- req.user 绑定并作为业务查询唯一来源
- 白名单路由放行，业务路由强制鉴权
- 统一错误码与错误结构（401/403/419）
- 角色与权限校验（RBAC）

### 安全与合规
- Token 密钥环境变量化并支持轮换
- Token 黑名单或 jti 机制支持主动登出
- 关键鉴权日志与审计日志

### 业务一致性
- 禁止使用前端传入的 userId 作为数据查询依据
- 资源级鉴权校验访问归属


## 请求缓存 已完成

- 后端为读多写少的接口增加缓存头（短时强缓存 + 可扩展协商缓存）
- 前端保留 GET 短时内存缓存作为兜底（避免短时重复请求）
- 对实时性强的接口（上传/识别/聊天）不缓存
- 缓存策略按业务域分级：dashboard、person > history > real-time
